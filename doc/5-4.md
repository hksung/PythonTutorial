# Frequency calculation

Text analysis often begins by counting how many times a particular linguistic unit occurs.

The results depend on the selected unit of analysis. In this tutorial, we will compare:

- **token frequency**, which counts surface forms;
- **lemma frequency**, which counts the lemmas assigned to those tokens.

For example, the English forms `write`, `writes`, `wrote`, and `writing` are different word-form tokens but may be grouped under the lemma `write`.

Subword frequencies answer a different question because subwords are model-specific computational units. For ordinary lexical-frequency analysis, we will therefore use the tokens and lemmas produced by spaCy.

## Using `collections.Counter`

Python's `Counter` class counts how many times each item occurs in a sequence.

```python
from collections import Counter

tokens = [
    "student",
    "write",
    "essay",
    "student",
    "write",
    "report"
]

frequency = Counter(tokens)

print(frequency)
print(frequency.most_common(3))
```

The output of `most_common()` is ordered from the most frequent item to the least frequent item.

## Loading the English and Korean pipelines

We will use the same spaCy pipelines introduced in the previous sections:

```python
import spacy
from collections import Counter

nlp_en = spacy.load("en_core_web_trf")
nlp_ko = spacy.load("ko_core_news_lg")
```

Use `nlp_en` for English texts and `nlp_ko` for Korean texts.

## Extracting tokens from a spaCy `Doc`

The following function extracts the surface form of each token:

```python
def extract_tokens(doc):
    return [
        token.text.lower()
        for token in doc
        if not token.is_space and not token.is_punct
    ]
```

This function:

1. retrieves each token using `token.text`;
2. converts it to lowercase;
3. excludes whitespace and punctuation.

## Extracting lemmas from a spaCy `Doc`

The following function extracts the lemma assigned to each token:

```python
def extract_lemmas(doc):
    return [
        token.lemma_.lower()
        for token in doc
        if not token.is_space and not token.is_punct
    ]
```

The token and lemma lists normally contain the same number of instances because each retained token receives one lemma. However, the lemma list may contain fewer unique types because several word forms can share the same lemma.

## English token and lemma frequencies

Consider the following English text:

```python
english_text = (
    "The student writes an essay. "
    "Yesterday, the student wrote a report. "
    "Other students are writing responses."
)

english_doc = nlp_en(english_text)
```

Extract the tokens and lemmas:

```python
english_tokens = extract_tokens(english_doc)
english_lemmas = extract_lemmas(english_doc)

print("English tokens:")
print(english_tokens)

print("\nEnglish lemmas:")
print(english_lemmas)
```

Calculate the frequencies:

```python
english_token_frequency = Counter(english_tokens)
english_lemma_frequency = Counter(english_lemmas)

print("Most common English tokens:")
print(english_token_frequency.most_common(10))

print("\nMost common English lemmas:")
print(english_lemma_frequency.most_common(10))
```

The word forms `writes`, `wrote`, and `writing` are counted separately in the token-frequency list. In the lemma-frequency list, they are grouped under the lemma `write`.

You can inspect the relationship directly:

```python
for token in english_doc:
    if not token.is_space and not token.is_punct:
        print(
            token.text,
            "→",
            token.lemma_
        )
```

## Korean token and lemma frequencies

Consider the following Korean text:

```python
korean_text = (
    "학생들은 과제를 작성하고 있었습니다. "
    "한 학생은 보고서를 작성했습니다. "
    "다른 학생들도 글을 작성하고 있습니다."
)

korean_doc = nlp_ko(korean_text)
```

Extract the tokens and lemmas:

```python
korean_tokens = extract_tokens(korean_doc)
korean_lemmas = extract_lemmas(korean_doc)

print("Korean tokens:")
print(korean_tokens)

print("\nKorean lemmas:")
print(korean_lemmas)
```

Calculate the frequencies:

```python
korean_token_frequency = Counter(korean_tokens)
korean_lemma_frequency = Counter(korean_lemmas)

print("Most common Korean tokens:")
print(korean_token_frequency.most_common(10))

print("\nMost common Korean lemmas:")
print(korean_lemma_frequency.most_common(10))
```

To understand which units are being counted, inspect the token, lemma, and fine-grained morphological tag together:

```python
for token in korean_doc:
    if not token.is_space and not token.is_punct:
        print(
            token.text,
            token.lemma_,
            token.tag_,
            sep="\t"
        )
```

For Korean, a spaCy token may contain multiple morphological elements. Therefore, a Korean lemma-frequency list counts the units returned by `token.lemma_`; it does not necessarily represent the frequency of individual morphemes or lexical stems.

## Counting instances and types

A **token instance** is one occurrence of a token in a text. A **token type** is a unique token form.

The length of the token list gives the number of token instances:

```python
print(
    "Number of English token instances:",
    len(english_tokens)
)

print(
    "Number of Korean token instances:",
    len(korean_tokens)
)
```

The length of a `Counter` gives the number of unique types:

```python
print(
    "Number of English token types:",
    len(english_token_frequency)
)

print(
    "Number of English lemma types:",
    len(english_lemma_frequency)
)

print(
    "Number of Korean token types:",
    len(korean_token_frequency)
)

print(
    "Number of Korean lemma types:",
    len(korean_lemma_frequency)
)
```

The number of lemma types may be smaller than the number of word-form types because different inflected forms can be grouped under the same lemma.

## A reusable frequency function

The following function calculates either token or lemma frequencies.

The language pipeline is passed to the function as an argument:

```python
def calculate_frequency(text, nlp, unit="lemma"):
    doc = nlp(text)

    if unit == "token":
        units = extract_tokens(doc)

    elif unit == "lemma":
        units = extract_lemmas(doc)

    else:
        raise ValueError(
            "unit must be either 'token' or 'lemma'"
        )

    return Counter(units)
```

Calculate English token and lemma frequencies:

```python
english_token_frequency = calculate_frequency(
    english_text,
    nlp_en,
    unit="token"
)

english_lemma_frequency = calculate_frequency(
    english_text,
    nlp_en,
    unit="lemma"
)

print(english_token_frequency.most_common(10))
print(english_lemma_frequency.most_common(10))
```

Calculate Korean token and lemma frequencies:

```python
korean_token_frequency = calculate_frequency(
    korean_text,
    nlp_ko,
    unit="token"
)

korean_lemma_frequency = calculate_frequency(
    korean_text,
    nlp_ko,
    unit="lemma"
)

print(korean_token_frequency.most_common(10))
print(korean_lemma_frequency.most_common(10))
```

The function is the same for both languages. However, the selected spaCy pipeline determines how the text is tokenized, morphologically analyzed, and lemmatized.

## Processing multiple documents

When processing several documents in the same language, use `nlp.pipe()` rather than applying `nlp()` separately to every text.

### English documents

```python
english_texts = [
    "The students write essays.",
    "One student wrote a report.",
    "Other students are writing responses."
]

english_lemma_frequency = Counter()

for doc in nlp_en.pipe(english_texts):
    english_lemma_frequency.update(
        extract_lemmas(doc)
    )

print(
    english_lemma_frequency.most_common(10)
)
```

### Korean documents

```python
korean_texts = [
    "학생들은 글을 씁니다.",
    "한 학생은 보고서를 썼습니다.",
    "다른 학생들은 답변을 쓰고 있습니다."
]

korean_lemma_frequency = Counter()

for doc in nlp_ko.pipe(korean_texts):
    korean_lemma_frequency.update(
        extract_lemmas(doc)
    )

print(
    korean_lemma_frequency.most_common(10)
)
```

`nlp.pipe()` is generally more efficient than calling `nlp()` repeatedly when processing many texts.

## Processing a multilingual corpus

Texts in different languages should first be separated by language and then processed using the corresponding pipeline.

```python
pipelines = {
    "en": nlp_en,
    "ko": nlp_ko
}

texts_by_language = {
    "en": [
        "The students wrote essays.",
        "They are revising their essays."
    ],
    "ko": [
        "학생들은 글을 썼습니다.",
        "학생들은 글을 수정하고 있습니다."
    ]
}
```

Calculate a separate lemma-frequency list for each language:

```python
lemma_frequencies = {}

for language, texts in texts_by_language.items():
    nlp_language = pipelines[language]
    frequency = Counter()

    for doc in nlp_language.pipe(texts):
        frequency.update(
            extract_lemmas(doc)
        )

    lemma_frequencies[language] = frequency
```

Inspect the results:

```python
print("English lemma frequencies:")
print(
    lemma_frequencies["en"].most_common(10)
)

print("\nKorean lemma frequencies:")
print(
    lemma_frequencies["ko"].most_common(10)
)
```

The English and Korean frequency lists should be interpreted separately.

A token or lemma produced by the English pipeline does not necessarily represent the same type of linguistic unit as a token or lemma produced by the Korean pipeline. Raw frequency counts should therefore not be compared directly across languages without a principled method for defining comparable units.

## Interpreting frequency results

Frequency calculation itself is straightforward, but the interpretation of the results depends on the preprocessing decisions made before counting.

Before interpreting a frequency list, consider:

- Was the frequency calculated using tokens or lemmas?
- Were capitalization differences removed?
- Were punctuation and whitespace excluded?
- Which language-specific pipeline was used?
- How did the pipeline define token boundaries?
- What does `token.lemma_` represent in that language?
- Are there possible tokenization, morphological-analysis, or lemmatization errors?
- Are the counted units comparable across languages?

Always report the exact spaCy pipeline and the unit used for frequency calculation.
